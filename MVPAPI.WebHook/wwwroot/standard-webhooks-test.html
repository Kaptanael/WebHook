Standard Webhooks Tester

POST /api/inbound using webhook-id, webhook-timestamp, webhook-signature headers. Signature is computed in-browser to match the server's StandardWebhookSigner.

Signing Setup

Standard Webhooks signs the request body with an HMAC-SHA256 key derived from your endpoint's SigningSecret. The signature is sent as three headers.

Endpoint Secret
Format: Detected as whsec_-prefixed (base64-encoded binary)
Message Headers
Unique message identifier. Auto-generated as msg_<random> if blank.
Current unix timestamp (seconds). Auto-generated if blank. Must not be >5 min in the past (300s tolerance).
Payload
Must be valid JSON. The exact bytes are signed, so no reformatting between signing and send.
Signature Computation
1. Secret Decoding
2. Signed Content (id.timestamp.body)
3. HMAC-SHA256 (hex)
4. Base64
5. Final Header Value
