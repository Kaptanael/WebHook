Standard Webhooks Tester

POST /api/inbound with X-Api-Key + webhook-id, webhook-timestamp, webhook-signature headers. Signature is computed in-browser to match the server's StandardWebhookSigner (keyed by whsec_+the key's Salt). Token-based tester: webhook-test.html

Signing Setup

Standard Webhooks signs the request body with an HMAC-SHA256 key derived from your endpoint's SigningSecret. The signature is sent as three headers.

API Key
Sent as X-Api-Key. The server keys the HMAC with whsec_+ that key's Salt — enter the Salt below.
Endpoint Secret
Format: Detected as whsec_-prefixed (base64-encoded binary)
Message Headers
Unique message identifier. Auto-generated as msg_<random> if blank.
Unix timestamp (seconds). Auto-generated if blank. Must be within 5 min (300s tolerance).
Payload
Optional. If the JSON body has a top-level type (Standard Webhooks envelope), that wins and this header is ignored — leave it blank. Otherwise the event type comes from here. Must resolve to a routed type (door.manual / relay.manual) to provision/deliver.
Exact bytes are signed — no reformatting between signing and send. For door.manual the body is delivered as-is to the door service: flat {companyId, actionType, strikeTimeOverrideInSeconds, doorIds} — actionType ∈ Unlock/Lock/UnlockHold, doorIds = reader GUIDs. This default is a verified end-to-end 200.
Signature Computation
1. Secret Decoding
2. Signed Content (id.timestamp.body)
3. HMAC-SHA256 (hex)
4. Base64
5. Final Header Value
