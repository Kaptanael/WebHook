MVPAPI WebHook Tester

Exercises POST /api/inbound (header-based auth) and POST /api/receiver (test sink). Served from wwwroot — same origin, so requests need no CORS.

Leave blank to call this app directly.

Inbound webhook → /api/inbound

The integration is resolved by the credential — no key in the URL. The matched endpoint's token (or signing secret) authenticates the request.

Signature is computed in-browser: v1,base64(HMAC-SHA256("id.ts.body", key)), matching StandardWebhookSigner (whsec_ secrets are base64-decoded for the key).


      

    

  


  
  

    
Receiver sink → /api/receiver

    
Local sink the dispatcher delivers to. Logs credential presence + payload and returns 200. Use it to verify outbound delivery shape.
